В этой лабораторной работе будет рассмотрен способ организации и упрощения кода программы с помощью методов. В первых заданиях вам нужно будет переписать с их помощью готовые решения задач из прошлых лабораторных. Об отличиях методов высокого качества от методов низкого качества можно прочитать в главе 7 "Высококачественные методы" части II книги "Совершенный код". (открыть в новой вкладке)

Подготовка проекта

Установите фреймворк для тестирования Jasmine версии 3. У вас должна получиться следующая структура папок:
root +-lib +-jasmine-3.?.? +-boot.js +-jasmine.css +-jasmine.js +-jasmine_favicon.png +-jasmine-html.js +-spec +-PlayerSpec.js +-SpecHelper.js +-src +-Player.js +-Song.js +-MIT.LICENSE +-SpecRunner.html
  • Папка root далее будет называться корнем директории.
  • Папка "src" предназначена для хранения файлов с исходным кодом.
  • Папка "spec" предназначена для хранения файлов с юнит-тестами.
  • Файл SpecRunner.html при открытии в браузере запускает проверку.
  • Создайте файл для исходного кода "lab12.js".
  • Создайте файл для юнит-тестов "lab12spec.js".
  • Подключите созданные файлы в SpecRunner.html.

Задача

Реализуйте функцию для решения задачи ЛР2#4411(открыть в новой вкладке)

Проверьте корректность работы функции с помощью вспомогательной программы

  1. В корне директории создайте файл Runner4411.html. Подключите в нем файл с кодом "src/lab12.js".
  2. В скриптах на странице Runner4411.html объявите переменную result.
  3. Для каждой строки тестовых данных из задачи напишите в программе фрагмент кода:
    1. Вызовите статический метод task4411 класса Methods.
    2. Передайте ему в качестве аргументов значения исходных данных для одного теста из задачи.
    3. Результат вызова метода запишите в переменную result.
    4. Напечатайте в консоль значение переменной result с точностью 4 знака после запятой.
  4. При открытии файла Runner4411.html, в консоли должен появиться текст:
18.1300 -7912.0000 7.0000

Проверьте корректность работы функции с помощью юнит-теста

  1. В файле с юнит-тестами вызовите функцию it. Она уже реализована в Jasmine. Первый ее аргумент - описание теста, например "в задаче 4411 должен получиться корректный результат". Второй аргумент - анонимная функция, содержащая код для проверки.
  2. В теле юнит-теста объявите переменную actual.
  3. Для каждой строки тестовых данных из задачи напишите в теле юнит-теста фрагмент кода:
    1. Вызовите функцию task4411.
    2. Передайте ей в качестве аргументов значения исходных данных для одного теста из задачи.
    3. Результат вызова функции запишите в переменную actual.
    4. Поскольку результат это число, то корректность результатов каждого вызова функции task4411 нужно проверить с помощью функции expect в сочетании с функцией toBeCloseTo. Они также реализованы в Jasmine.
    5. Вызовите функцию expect и передайте ей в качестве аргумента переменную actual.
    6. В той же строке через точку вызовите функцию toBeCloseTo. Первый аргумент этой функции - ожидаемое значение. Второй - точность. Это количество знаков после запятой, которые должны совпадать в результате. Для этой задачи достаточно указать 4.
  4. При открытии файла SpecRunner.html, в нем должна быть зеленая строка "в задаче 4411 должен получиться корректный результат".
