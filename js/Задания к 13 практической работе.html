В данной лабораторной работе внимание уделяется оформлению исключений в изучаемом языке программирования. Однако использование исключений не всегда оправдано. Полезные советы о том, когда лучше использовать механизм исключений, содержатся в главе 8.4 "Исключения" части II книги "Совершенный код". (открыть в новой вкладке)

9020

Задача

  1. В файле для исходного кода "lab13.js" реализуйте функцию task9020 для решения задачи ЛР3#9020(открыть в новой вкладке).
  2. В теле функции task9020 нужно предусмотреть обработку некорректных исходных данных.
  3. Если в качестве аргументов были переданы некорректные значения, нужно создать исключение и "выбросить" его с помощью ключевого слова throw. При создании исключения, в скобках нужно указать текст ошибки.
  4. Если все аргументы корректны, нужно вычислить результат и вернуть его.
  5. В файле для юнит-тестов "lab13spec.js" проверьте с помощью юнит-теста результат работы функции task9020 в случаях с корректными данными.
  6. Проверьте с помощью вспомогательного файла Runner9020.html результат работы функции task9020 в случаях с корректными данными.
23.4307 25.0998 0.0000 3.0000

Проверьте юнит-тестами, что функция выбрасывает исключение при некорректных данных

  1. Чтобы убедиться, что функция выбрасывает исключение, нужно воспользоваться функцией expect в сочетании с функцией toThrow.
  2. В теле юнит-теста создайте экземпляр ошибки, который будет "эталоном" для сравнения.
  3. Сначала нужно вызвать функцию expect и передать ей в качестве аргумента анонимную функцию, в теле которой нужно вызвать функцию task9020 с некорректным значением аргумента.
  4. В конце той же строки нужно поставить точку и вызвать функцию toThrow. Единственный ее аргумент - эталон ошибки.
  5. Всего таких конструкций должно получиться 3. По одной для каждого некорректного значения.
  6. При открытии файла SpecRunner.html, в нем должна быть зеленая строка "в задаче 9020 должен получиться корректный результат".

Добавьте во вспомогательный файл Runner9020.html обработку исключений

  1. Вызовите функцию task9020 с аргументами из каждого теста, указанного в задаче.
  2. В случае успешного выполнения функции task9020, выведите в консоль результат с точностью до 4 знаков после запятой.
  3. В случае возникновения исключения, нужно его перехватить и вывести в консоль сообщение об ошибке.
  4. При открытии файла Runner9020.html, в консоли должен появиться текст:
23.4307 25.0998 0.0000 3.0000 Подкоренное выражение должно быть неотрицательным Подкоренное выражение должно быть неотрицательным Подкоренное выражение должно быть неотрицательным
